# ProofRail Silver Multi-Agent Harness Run Report Schema v0.1.0

**Version:** v0.1.0
**Date:** 2026-06-21
**Status:** Draft / Demo-informed schema
**Claim family:** ProofRail Silver multi-agent attack harness run reports

---

## 1. Purpose

The Silver Multi-Agent Harness Run Report records the result of a single harness execution. It enumerates per-event outcomes, summarizes counts, and records the canonical non-execution invariant.

A run report is generated by `tools/silver/run_multi_agent_attack_harness_v0_1_0.py` and consumed by:

- `tools/silver/verify_multi_agent_harness_evidence_v0_1_0.py` (via the evidence manifest);
- regression tests;
- human inspection.

The run report is **not**:

- a Silver Verification Report;
- a Silver Profile Conformance Report;
- a Verifier Output Attestation subject;
- a signed certification artifact.

---

## 2. Relationship to v0.2.3 Decision Reports

For every `protected_action_attempt` event, the harness invokes the v0.2.3 authority evaluator and writes a Silver Protected Action Decision Report v0.1.0 to `authority-decision-reports/<event_id>.json`. The run report references each such decision report by output path.

The run report does **not** duplicate decision report content. The decision report files remain the authoritative authority artifacts and are hashed separately by the evidence manifest.

---

## 3. Format

JSON. UTF-8 encoded. Serialized with `json.dumps(obj, indent=2, sort_keys=True)` followed by a trailing newline for deterministic output.

---

## 4. Top-Level Structure

```json
{
  "report_type": "proofrail.silver.multi_agent_harness_run_report",
  "report_version": "v0.1.0",
  "script_id": "<string>",
  "authority_fixture_id": "<string>",
  "started_at": "<ISO-8601>",
  "completed_at": "<ISO-8601>",
  "summary": { ... },
  "event_results": [ ... ],
  "outputs": { ... },
  "execution": { ... },
  "limitations": [ ... ]
}
```

All top-level fields are required.

| Field | Notes |
|---|---|
| `report_type` | Must equal `proofrail.silver.multi_agent_harness_run_report`. |
| `report_version` | Must equal `v0.1.0`. |
| `script_id` | Copied from the harness script. |
| `authority_fixture_id` | Copied from the authority fixture. |
| `started_at` | Wall-clock ISO-8601 UTC start. Documentation only; not part of any cryptographic guarantee. |
| `completed_at` | Wall-clock ISO-8601 UTC completion. |
| `summary` | Aggregate counts and overall status. |
| `event_results` | Ordered per-event results. |
| `outputs` | Relative paths to per-event output files. |
| `execution` | Top-level non-execution invariant. |
| `limitations` | Non-empty list of non-claim strings. |

---

## 5. Summary

```json
"summary": {
  "status": "pass" | "fail",
  "events_total": <int>,
  "events_passed": <int>,
  "events_failed": <int>,
  "protected_action_attempts": <int>,
  "actions_allowed": <int>,
  "actions_denied": <int>,
  "bypass_attempts_blocked": <int>,
  "revocation_markers": <int>
}
```

`status` is `pass` if and only if every event's actual outcome matched its `expected.harness_outcome` (and, for `protected_action_attempt`, `decision_status` and `decision_reason`).

---

## 6. Event Results

Each entry in `event_results`:

```json
{
  "event_id": "<string>",
  "event_type": "<string>",
  "timestamp": "<ISO-8601>",
  "from_principal_id": "<string>",
  "to_principal_id": "<string>",
  "actual": {
    "harness_outcome": "<enum>",
    "decision_status": "<allow|deny|null>",
    "decision_reason": "<string|null>",
    "harness_reason": "<string|null>"
  },
  "expected": {
    "harness_outcome": "<enum>",
    "decision_status": "<allow|deny|null>",
    "decision_reason": "<string|null>",
    "harness_reason": "<string|null>"
  },
  "match": true | false,
  "outputs": {
    "request_path": "<relative-path|null>",
    "decision_report_path": "<relative-path|null>"
  }
}
```

For `agent_message`, `bypass_attempt`, and `revocation_marker` events, `actual.decision_status` and `actual.decision_reason` are `null`.

For `protected_action_attempt` events, `actual.harness_reason` is `null`.

---

## 7. Outputs

```json
"outputs": {
  "transcript_path": "transcript.jsonl",
  "requests_dir": "protected-action-requests",
  "decision_reports_dir": "authority-decision-reports",
  "expected_outcomes_path": "expected-outcomes.json"
}
```

All paths are relative to the harness output directory.

---

## 8. Execution Invariant

```json
"execution": {
  "protected_actions_performed": false,
  "reason": "harness_decision_only"
}
```

`protected_actions_performed` **must** be `false`. The verifier rejects any run report with this set to `true` as `execution_violation`.

---

## 9. Limitations

The `limitations` list must be non-empty. Recommended entries:

```
Local deterministic harness execution only.
No live agents executed.
No live actuators invoked.
Not natural-language prompt parsing.
Not prompt-injection detection.
Not Gold certification.
```
